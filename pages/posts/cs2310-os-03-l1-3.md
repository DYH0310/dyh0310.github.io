# Lecture 1-3 指令集体系结构（ISA）与 MIPS 汇编

- 来源：`L1-3-体系结构.pptx.pdf`
- 目标：补齐理解操作系统所需的最小硬件/汇编背景，重点理解指令、寄存器、存储器与编码格式。

## 本讲主线

这一讲不是要求你会写复杂汇编，而是要你理解：

1. 程序最终一定会落到机器指令。
2. 操作系统管理进程、地址空间、中断，本质上都依赖 ISA 暴露的能力。
3. MIPS 作为经典 RISC 架构，适合用来学习“简单、规整、可编码”的设计思想。

## 知识点 1：机器语言与汇编语言

- 机器语言是 CPU 可以直接执行的二进制指令序列。
- 汇编语言是机器指令的人类可读助记符表示。
- 不同 ISA 的汇编语言通常不同，因此汇编程序通常不具备跨平台能力。

例子：

- 同样表示加法，MIPS 可能写成 `add $s0, $s1, $s2`，但换到 x86 就会是完全不同的写法。
- C 程序可跨平台重编译，而汇编程序通常和具体处理器绑定。

## 知识点 2：编译系统把高级语言转成可执行代码

- 高级语言不会被 CPU 直接执行，必须经过预处理、编译、汇编、链接等阶段。
- 编译阶段把高级语言变为汇编或中间表示。
- 汇编阶段把汇编代码变成目标文件中的机器码。
- 链接阶段把多个目标文件和库组合成最终可执行文件。

例子：

- 一段 `int a = b + c;`，最终会被转换成若干条寄存器操作与可能的访存指令。
- 一个程序调用 `printf()` 时，链接阶段会把标准库中的符号解析到程序中。

## 知识点 3：MIPS 与 RISC 设计思想

- MIPS 是经典的 RISC（精简指令集）架构代表。
- RISC 强调指令简单、格式规整、便于硬件高效实现。
- 讲义中强调的设计准则包括：简单设计有助于规整化、加速常用功能、越小越快、设计需要折中。

例子：

- MIPS 不追求一条指令做太多复杂工作，而是倾向于用多条简单指令完成任务。
- `a = b + c - d` 可以拆成两条指令：先 `add`，再 `sub`。

## 知识点 4：操作数的三种主要来源

- 寄存器：速度最快、数量少，适合保存热点数据。
- 存储器：容量大、速度慢，适合保存大量数据。
- 立即数：直接写在指令里，适合小常量。

例子：

- `add $t0, $t1, $t2` 的操作数都来自寄存器。
- `addi $t0, $t1, 5` 里的 `5` 是立即数。
- `lw $s0, 4($sp)` 需要从内存中读取数据。

## 知识点 5：寄存器是最快但最稀缺的计算资源

- MIPS 有固定数量的通用寄存器，讲义中强调 MIPS 是 32 位体系结构。
- 寄存器比内存访问快很多，因此编译器会尽量把常用变量放进寄存器。
- 但寄存器数量有限，放不下的数据必须溢出到内存。

例子：

- 循环计数器、函数局部热点变量通常优先驻留寄存器。
- 当函数调用层级变深时，一些寄存器内容可能要先保存到栈里。

## 知识点 6：MIPS 是典型的 Load/Store 架构

- 算术逻辑运算通常只在寄存器之间进行。
- 从内存读取数据要用 `lw`，写回内存要用 `sw`。
- 这意味着“访存”和“运算”被清晰分离。

例子：

- 如果要计算 `x = y + z`，而 `y` 和 `z` 在内存里，通常步骤是：先 `lw` 读入寄存器，再 `add` 运算，最后必要时 `sw` 写回。

## 知识点 7：基址 + 偏移是常见的寻址方式

- `lw rt, imm(rs)` 表示访问地址 `rs + imm` 上的数据。
- 这种设计非常适合访问数组、结构体字段和栈帧中的局部变量。
- 操作系统中的很多底层实现，如切换栈、访问 PCB 字段，也依赖类似寻址思想。

例子：

- `lw $s0, 8($sp)` 可以理解为“从当前栈顶基址往下偏移 8 字节处取一个字”。
- 访问结构体成员时，编译器也常把“对象首地址 + 字段偏移量”编译成这种形式。

## 知识点 8：R、I、J 三类基本指令格式

### 8.1 R 型指令

- 主要用于寄存器之间的运算。
- 典型字段包括 `op`、`rs`、`rt`、`rd`、`shamt`、`funct`。
- `op` 常指明大类，`funct` 进一步确定具体操作。

例子：

- `add $s0, $s1, $s2` 表示把 `$s1` 和 `$s2` 相加后写入 `$s0`。
- `sub $t0, $t3, $t5` 表示寄存器减法。

### 8.2 I 型指令

- 用于带立即数的运算或访存指令。
- 典型字段包括 `op`、`rs`、`rt`、`imm`。

例子：

- `addi $s0, $s1, 5` 表示 `$s1 + 5` 写入 `$s0`。
- `lw $t2, 32($0)` 表示从地址 `32` 处读取一个字到 `$t2`。

### 8.3 J 型指令

- 用于跳转类操作。
- 通过操作码和目标地址字段控制程序流。

例子：

- `j target` 可用于无条件跳转到某一段代码。

## 知识点 9：逻辑运算与移位是底层程序的重要工具

- 常见逻辑运算包括 `and`、`or`、`xor`、`nor`。
- 常见移位操作包括 `sll`、`srl`、`sra` 以及变量移位版本。
- 很多系统程序会用逻辑与移位做位标志处理、权限编码、地址对齐和掩码操作。

例子：

- 判断某个权限位是否开启，可以通过 `and` 与掩码配合完成。
- `sll` 左移 2 位可以近似看成乘 4，常用于数组索引地址计算。

## 知识点 10：为什么这部分内容对操作系统重要

- 进程切换需要保存和恢复寄存器。
- 中断处理依赖 PC、栈和特权切换。
- 地址空间管理依赖访存模型和地址翻译机制。
- 系统调用进入内核时，本质上也是按 ISA 规定的机制改变控制流。

例子：

- 操作系统切换线程时，不是“神奇地切换”，而是把当前线程寄存器状态存到某处，再把另一个线程的寄存器状态装回来。
- 内核处理系统调用后返回用户态，本质上也是恢复先前保存的执行上下文。

## 这讲结束后应该形成的认知

- 汇编不是独立于操作系统的知识，而是理解内核执行机制的最小底层语言。
- 只要后面看到“寄存器、栈、PC、上下文切换、访存、特权级”，都应该能联想到这里的 ISA 基础。
