# Lecture 1-1 操作系统概述

- 来源：`L1-1-操作系统概述-3.pptx.pdf`
- 目标：建立“什么是操作系统、操作系统为什么存在、后续课程学什么”的总框架。

## 本讲主线

这一讲的核心不是记定义，而是建立一个总视角：

1. 计算机系统由硬件、操作系统、应用程序、用户共同构成。
2. 操作系统既是资源管理者，也是抽象提供者。
3. 现代操作系统的关键问题包括进程、内存、I/O、文件、保护、安全和虚拟化。
4. 操作系统的发展历史，决定了今天常见系统设计的很多思想来源。

## 知识点 1：计算机系统的组成与分层

- 计算机系统通常可以分为四层：硬件、操作系统、应用程序、用户。
- 硬件负责提供基础计算能力，例如 CPU、主存、磁盘、网络设备。
- 操作系统位于硬件和应用之间，对上提供统一接口，对下管理复杂硬件。
- 应用程序不直接控制硬件，而是通过操作系统提供的接口完成计算、存储和通信。

例子：

- 浏览器需要读取网页缓存时，并不是自己直接向 SSD 发命令，而是调用文件系统接口，由操作系统安排块设备访问。
- 视频播放器播放电影时，同时使用 CPU 解码、内存缓存、显卡显示、声卡输出；这些资源的协调由操作系统完成。

## 知识点 2：操作系统的两种核心角色

### 2.1 资源管理者

- 操作系统负责管理 CPU、内存、外设、文件和网络等资源。
- 多个程序同时运行时，资源会竞争，操作系统必须决定“谁先用、能用多少、何时释放”。
- 这意味着操作系统本质上要解决分配、复用、隔离和回收的问题。

例子：

- 3 个进程同时请求 CPU，操作系统需要通过调度算法决定当前先运行哪一个。
- 两个程序都要打印文件时，操作系统需要排队管理打印机，避免设备被并发写乱。

### 2.2 抽象提供者

- 操作系统向应用提供比硬件更高层的抽象，例如“进程”“文件”“虚拟内存”“套接字”。
- 这些抽象隐藏了硬件细节，让应用程序更容易编写，也更容易移植。
- 程序员通常面向操作系统抽象编程，而不是面向裸硬件编程。

例子：

- 程序员打开一个文件时只关心路径和读写权限，不需要知道文件究竟落在磁盘的哪几个物理扇区。
- 程序员创建一个进程时看到的是 PID、地址空间、文件描述符，而不是具体寄存器布线细节。

## 知识点 3：从 Hello World 看操作系统到底做了什么

- 一个看似简单的 `Hello World`，背后会经历编译、链接、装载、创建进程、分配地址空间、执行系统调用、输出到终端等过程。
- 程序运行前，编译系统把高级语言转换成目标代码并链接成可执行文件。
- 程序启动时，操作系统会创建进程、建立页表、准备栈和堆，并把程序装入内存。
- `printf()` 这类输出最终会走到系统调用，再由内核与终端/控制台驱动交互。

例子：

- `printf("Hello World\n");` 表面上是函数调用，但底层通常会落到类似 `write()` 的系统调用，再由内核把字符送到终端设备。
- 双击一个可执行文件时，用户只看到“程序打开了”，操作系统实际上已经完成了装载、权限检查、内存映射和调度准备。

## 知识点 4：从系统视角理解操作系统

- 从系统视角看，操作系统关注的是如何让整台机器高效、稳定、公平地运行。
- 关键目标包括：提高资源利用率、提升吞吐量、减少冲突、保证隔离、控制错误传播。
- 操作系统需要管理应用的生命周期：创建、运行、阻塞、唤醒、终止。

例子：

- 如果某个程序频繁进行磁盘 I/O，操作系统可能在它等待磁盘时切换去执行另一个程序，以避免 CPU 空转。
- 如果一个程序崩溃，操作系统需要尽量把影响限制在该进程内，而不是拖垮整个系统。

## 知识点 5：从用户视角理解操作系统

- 从用户视角看，操作系统的价值在于“易用、快、稳、安全”。
- 用户希望同一台机器可以同时开浏览器、编辑器、播放器，并且感觉每个程序都“在为自己服务”。
- 时间共享系统让多个用户或多个程序看起来像独占系统一样工作。

例子：

- 用户在打字时，希望按键马上回显；这体现的是响应时间。
- 用户复制大文件时，希望系统不要完全卡死；这体现的是调度和 I/O 管理的平衡。

## 知识点 6：中断是操作系统获得控制权的关键机制

- 如果 CPU 一直顺序执行用户程序，操作系统就无法重新接管系统。
- 中断和异常使 CPU 可以在合适时刻切换到内核，处理设备事件、错误或系统调用请求。
- 中断来源可以是外部设备，也可以是时钟，也可以是程序执行中的异常情况。

例子：

- 键盘按下一个键会触发键盘中断，操作系统据此读取输入并交给前台程序。
- 时钟中断周期性发生，操作系统可以借此进行时间片轮转，强制切换进程。
- 除零错误会触发异常，操作系统可以终止出错进程并给出错误信息。

## 知识点 7：计算机系统体系结构决定了操作系统的工作边界

- 操作系统不是凭空存在的，它必须建立在具体硬件体系结构之上。
- 单处理器、多处理器、多核、集群系统，对操作系统提出的管理问题不同。
- 体系结构变化会直接影响调度、同步、缓存一致性、I/O 管理和可扩展性。

例子：

- 单核系统中的“并发”主要靠时间片轮转；多核系统中的“并发”则可能是真正同时执行。
- 集群系统中，多个节点通过网络协作，操作系统或集群管理软件需要处理节点之间的通信与容错。

## 知识点 8：操作系统发展的历史主线

- 早期系统更强调批处理，把作业按批次送入机器执行。
- 后来出现多道程序设计，让多个作业交替推进，提高 CPU 利用率。
- 再后来，多用户分时系统强调交互性和响应速度。
- UNIX 体系把“文件、一切皆可组合、工具化”思想推广开来，Linux 在此基础上成为现代主流系统之一。
- PC 体系上的 DOS、Windows、Mac OS、iOS 展示了另一条以个人计算和图形界面为中心的发展路径。

例子：

- 批处理时代，一个作业跑完另一个再开始，CPU 很容易在 I/O 等待中空闲。
- 分时系统允许多个用户共享主机，每个人都感觉自己在独占使用机器。
- UNIX 中可以把多个小工具通过管道串起来，这种“组合式”思想后来深刻影响了系统设计。

## 知识点 9：本课程的整体知识地图

- 进程管理：研究程序如何变成可调度的执行实体。
- 线程与并发：研究一个进程内如何拆分执行流，以及多执行流如何协作。
- CPU 调度：研究多个可运行任务如何竞争处理器。
- 同步与死锁：研究共享数据和资源竞争下如何保证正确性。
- 内存管理：研究地址空间、分页、虚拟内存和缓存等问题。
- 存储管理：研究文件系统、I/O 设备、大容量存储器。
- 保护与安全：研究权限、隔离、可信执行环境。
- 虚拟化：研究如何把一套硬件抽象成多套“看似独立”的运行环境。

例子：

- 一个 Web 服务器会同时涉及进程/线程、CPU 调度、套接字、文件系统、同步原语和安全权限，几乎覆盖整门课的大部分主题。

## 这讲结束后应该形成的认知

- 不要把操作系统只理解成“Windows/Linux 这种软件”。
- 更准确的理解是：操作系统是一组管理资源、提供抽象、维持执行环境的机制和策略。
- 后续所有章节，本质上都在回答一个问题：如何让大量程序安全、高效、可控地共享同一套硬件。
